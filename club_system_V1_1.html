Phinma COC Club Verification System

Java Console Application - Project Documentation

A comprehensive club management system built as a Java Console Application for PHINMA COC students, featuring menu-driven interfaces for both Administrator and Student users with real-time data management using ArrayList data structures.

System Overview

What is this system?

This is a School Club Registration and Verification System Using Student ID designed for PHINMA COC to streamline the process of student club membership and administration. Built as a Java Console Application, it provides a menu-driven interface for both students and administrators.

Main Features:

  • Student Registration with Validation: Students register using 11-digit numeric ID, Full Name (letters/spaces/hyphens/apostrophes/periods only, minimum 2 characters), and Year Level selection (1st-4th Year)
  • Default Department Assignment: All students are automatically assigned to CITE department upon registration
  • Club Membership: Join from 5 different school clubs: Basketball, Volleyball, Chess, Badminton, or Billiards
  • Status-Based Workflow: Dynamic system that adapts to current status - No Club → Pending → Active, with Pending Leave option for active members
  • Admin Approval System: Administrators verify join requests and process leave requests through a dedicated dashboard
  • Activity Announcements: Admins can post school activities and events visible to all students

Administrator Dashboard (4 Main Functions)

  • 1. Verify JOIN Requests: Review pending student applications, view their details (ID, Name, Year, Department, Requested Club), and approve or reject membership
  • 2. Verify LEAVE Requests: Process pending leave requests, review the student's reason for leaving, and allow or deny the request
  • 3. View Members: Generate complete membership report showing all active students with their ID, Name, Year, Department, and Club affiliation
  • 4. Post Activity/Event: Update the school announcement that all students can view from their dashboard

Student Dashboard (3 Main Options)

Menu Options Available to Students:

  1. Register to a Club: Status-dependent functionality:
    • If No Club: Browse 5 available clubs and submit join request
    • If Pending: Change club selection or cancel current request
    • If Active: Option to leave current club (requires reason)
    • If Pending Leave: Cancel leave request or wait for admin approval
  2. Check Verification Status: View complete profile including Student ID, Name, Year Level, Department, and current status:
    • No Club: Not a member of any club
    • Pending: Waiting for admin verification on club request
    • Active: Verified member of a specific club
    • Pending Leave: Leave request submitted, waiting for admin decision
  3. View Activities/Events: Display current school announcement posted by administrator

Detailed Features

1. Authentication & Registration System

Admin Login Process:

  • 3 Predefined Admin Accounts: admin/admin123, prof/prof123, teach/teach123
  • Username/Password Validation: Case-insensitive username check, exact password match
  • Exit Options: Type 'exit' to quit program or 'back' to return to user type selection

Student Registration/Login Process:

  1. 11-Digit Student ID Validation:
    • Must be exactly 11 characters long
    • Must contain ONLY numeric digits (no letters or symbols)
    • Error messages show exact character count entered
  2. Full Name Validation:
    • Cannot be empty
    • Minimum 2 characters required
    • Numbers are NOT allowed
    • Special characters allowed: spaces, hyphens (-), apostrophes ('), periods (.)
  3. Year Level Selection (Menu):
    • Option 1: 1st Year
    • Option 2: 2nd Year
    • Option 3: 3rd Year
    • Option 4: 4th Year
    • Option 0: Go Back
  4. Automatic Account Creation:
    • New students: Account created with status "No Club", department set to "CITE"
    • Returning students: Name must match registered name exactly (case-insensitive)
    • Default password for all students: 1234

2. Student Dashboard - Status-Based Logic

When student chooses "Register to a Club":

Status: No Club (or null)

  • Display list of 5 clubs: Basketball, Volleyball, Chess, Badminton, Billiards
  • Student selects club by number (1-5) or enters 0 to go back
  • System updates: Status → "Pending", RequestedClub → selected club
  • Shows confirmation with full student details and "Waiting for admin verification..." message

Status: Pending

  • Shows current pending request
  • Options:
    1. Change Club: Select different club from list, updates requested club
    2. Cancel Request: Status reverts to "No Club", requested club cleared
    3. Go Back: Return to main menu

Status: Active

  • Shows current verified club membership
  • Option to leave club:
    1. Yes: Must enter reason (cannot be empty), Status → "Pending Leave"
    2. No: Return to dashboard
    3. Go Back: Return to main menu

Status: Pending Leave

  • Shows current club and pending leave status
  • Options:
    1. Cancel Request: Status reverts to "Active", leave reason cleared
    2. No: Keep waiting for admin approval
    3. Go Back: Return to main menu

3. Administrative Panel - Exact Workflow

Option 1: Verify JOIN Requests

  1. System displays all students with status "Pending" showing: ID, Name, Year, Department, Requesting Club
  2. Admin enters Student ID to process (or presses Enter/0 to go back)
  3. If ID found with pending request:
    • Approve (1): Status → "Active", JoinedClub = RequestedClub, RequestedClub cleared
    • Reject (2): Status → "No Club", RequestedClub cleared
    • Go Back (0): Return to dashboard, request remains pending

Option 2: Verify LEAVE Requests

  1. System displays all students with status "Pending Leave" showing: ID, Name, Current Club, Reason
  2. Admin enters Student ID to process (or presses Enter/0 to go back)
  3. If ID found with pending leave:
    • Allow (1): Status → "No Club", JoinedClub cleared, LeaveReason cleared
    • Deny (2): Status → "Active", LeaveReason cleared (student remains in club)
    • Go Back (0): Return to dashboard, request remains pending

Option 3: View Members

  • Displays all students with status "Active"
  • Shows complete profile: ID, Name, Year, Department, Club
  • Message "No active members found" if list is empty

Option 4: Post Activity/Event

  • Displays current announcement
  • Admin enters new announcement text (or 0 to go back)
  • Validation: Announcement cannot be empty
  • Updates global announcement variable visible to all students

Key Java Implementation Details:

  • Console I/O with Scanner: All user input through scanner.nextLine() and scanner.nextInt()
  • Comprehensive Input Validation:
    • Numeric validation for Student ID (Character.isDigit check)
    • Name validation (no numbers, allowed special characters, minimum length)
    • Empty field checks with error messages
    • InputMismatchException handling for numeric inputs
  • Session Management: Boolean flags control program flow:
    • menuRunning: Main program loop
    • userRunning / adminRunning: User-specific dashboard loops
    • isStudent: Distinguishes student vs admin session
    • currentUserIndex: Tracks current user's position in ArrayLists (-1 if not logged in)
  • Menu-Driven Interface: Switch-case statements for all user choices with default error handling
  • Status State Machine: Four-state workflow (No Club → Pending → Active → Pending Leave → No Club)
  • Dynamic Account Handling: New students auto-created on first login, returning students verified by name match

Code Highlights

Example 1: 11-Digit Student ID Validation

// Validate 11-digit Student ID - Length Check
final int validIdLength = 11;

if (inputUsername.length() != validIdLength) {
    System.out.println("ID must be exactly " + validIdLength + 
        " digits! You entered: " + inputUsername.length() + " character(s).");
    continue;
}

// Numeric-only validation using Character.isDigit()
boolean isNumeric = true;
for (char c : inputUsername.toCharArray()) {
    if (!Character.isDigit(c)) {
        isNumeric = false;
        break;
    }
}

if (!isNumeric) {
    System.out.println("ID must contain ONLY numbers (no letters/symbols).\n");
    continue;
}

Example 2: Full Name Validation (Letters & Select Special Characters Only)

// Name validation - Step 1: Check for numbers
boolean nameHasNumbers = false;
for (char c : fullName.toCharArray()) {
    if (Character.isDigit(c)) {
        nameHasNumbers = true;
        break;
    }
}
if (nameHasNumbers) {
    System.out.println("Name must contain ONLY letters (numbers are NOT allowed).\n");
    continue;
}

// Name validation - Step 2: Check for invalid special characters
boolean nameHasInvalidChars = false;
for (char c : fullName.toCharArray()) {
    // Allow only: letters, spaces, hyphens, apostrophes, periods
    if (!Character.isLetter(c) && !Character.isWhitespace(c) 
        && c != '-' && c != '\'' && c != '.') {
        nameHasInvalidChars = true;
        break;
    }
}
if (nameHasInvalidChars) {
    System.out.println("Special characters (except - ' .) are not allowed.\n");
    continue;
}

Example 5: Dynamic Student Registration - New vs Returning User

// Search for existing student by ID
int foundIndex = -1;
for (int i = 0; i < studentIds.size(); i++) {
    if (studentIds.get(i).equals(inputUsername)) {
        foundIndex = i;
        break;
    }
}

if (foundIndex == -1) {
    // NEW STUDENT - Create account
    loginSuccess = true;
    isStudent = true;
    currentLoggedInName = fullName;
    
    // Add to all ArrayLists with default values
    studentIds.add(inputUsername);
    studentStatus.add("No Club");
    studentJoinedClub.add(null);
    studentRequestedClub.add(null);
    studentLeaveReason.add(null);
    studentNames.add(fullName);
    studentYears.add(yearStr);
    studentDepartments.add(DEFAULT_DEPARTMENT);  // "CITE"
    currentUserIndex = studentIds.size() - 1;
    
    System.out.println("\nAccount created successfully!");
    System.out.println("Welcome, " + fullName + "!");
    
} else {
    // RETURNING STUDENT - Verify name matches records
    String storedName = studentNames.get(foundIndex);
    
    if (!storedName.equalsIgnoreCase(fullName)) {
        System.out.println("\nName does not match our records!");
        System.out.println("Registered Name: " + storedName);
        System.out.println("Entered Name: " + fullName);
        System.out.println("Login denied.");
        loginSuccess = false;
        
    } else {
        // Name matches - Login successful
        loginSuccess = true;
        isStudent = true;
        currentLoggedInName = storedName;
        currentUserIndex = foundIndex;
        studentYears.set(currentUserIndex, yearStr);  // Update year level
        
        System.out.println("\nWelcome back, " + storedName + "!");
    }
}
